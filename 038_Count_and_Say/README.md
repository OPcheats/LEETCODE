 Problem Statement
The count-and-say sequence is a sequence of digit strings defined as:

- countAndSay(1) = "1"
- countAndSay(n) is the run-length encoding (RLE) of countAndSay(n - 1)

Given a positive integer `n`, return the nth term of the count-and-say sequence.

---

 Core Idea
Each term is generated by **describing the previous term**.

We read the previous string and:
- Count consecutive identical digits
- Say the count followed by the digit

This is exactly **Run-Length Encoding (RLE)**.
